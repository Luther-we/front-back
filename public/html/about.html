Ce jeu a été développé dans le cadre d’un projet d’école Front / Back.

Le Front, en dehors du HTML et CSS, est une single page application à 98% en JavaScript natif, les 2 pour-cent restant étant jQuery.
Le Back repose sur une solution Socket.IO / NodeJS.
Certains parlent d’une Beta, selon moi on est proche de la Delta (v0.0.0.1).
Le bilan après cela est clair :

  • mise en place de Framework REACT pour le prochain projet, pour sa fabuleuse fonction render()
  • exploser le script client entre les Emit/On de Socket.IO et les fonctions
  • garder le même couple NodeJS / Socket.IO mais moduler le code entre les fonctions d’appel et les comportements de jeu
  • microtester les fonctions et les éléments via un dossier/serveur template pour ne pas se noyer sous un code trop lourd et un débogage trop riche
  • créer une véritable console de débogage
  • faire sa mise en page au fur et à mesure
  • commencer tôt pour ne pas implémenter la moindre fonction à la va-vite et oublier les options
  • limiter les valeurs écrites au profit de fonction return
  • arrêter de s’amuser avec le nom des variables (franglais, références musicales... )
  • une fonction = une seule chose exécutée !!!
  • …
